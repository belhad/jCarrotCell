Horizontal Scroller

The default carrotcell is horizontal and only needs a few simple steps to set up. See the home page for the instructions.

prev
next

The default carrotcell needs to be invoked on the .carrotCell container to start working.


$('#defaultKitten').carrotCell();

Make sure you have styles for the .carrotCell container and its child elements, including a fix width view port and fixed same width cell elements.


// insert your own styling for the carousel elements
// including .carrotCell and .carrotCellView width and height
// and .carrotCellView li width and height - these must all be the same size for the calculations to work
// also include the placement of the previous and next buttons
// below are examples
	
#defaultKitten { width: 560px; }
.carrotCellView li { ... } 
.carrotCellView li img { ... }
.carrotCell .next, .carrotCell .prev {
	position: absolute;
	top: 60px;
	display: none;
	cursor: pointer;
	...
}
...

The basic structure of the carrotcell HTML must be followed exactly in order for it to work correctly in basic setup. See the Layout Flexibility demo to find out how to change this mark up.

Vertical Scroller

This carrotCell scrolls vertically instead of horizontally.

prev next

For a vertical carrotCell you must set the parameter sideways to be the boolean false


$('#defaultVerticalKitten').carrotCell({ sideways: false });

You must create the styles to lay out the .carrotCell and children correctly, this is not included in the .carrotCell.css.


#defaultVerticalKitten { width: 180px; }
#defaultVerticalKitten .carrotCellView { height: 260px; }
...

The mark up used in the vertical carrotCell is exactly the same as the horizontal one, the only difference is your changes in the CSS.

Fall Back State (No Javascript)

This version uses the overflow:auto property of the .carrotCellView container when Javascript is not available. You can also over ride it to just be really long if you wish.
