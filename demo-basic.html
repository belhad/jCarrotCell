Getting Started

1. Create the carrotCell markup. By default everything must be contained in the element .carrotCell, which contains a .carrotCellView that is the parent to another container with the items being its children. The controls are also assumed to be contained inside .carrotCell. The markup can be changed, please refer to the layout flexibility demo and the documentation.

2. Include the carrotcell.css or import the styles into your own style sheet.


<link type="text/css" href="css/carrotcell.css" rel="stylesheet">

3. Include the jQuery javascript file (last tested with 1.7.2).


<script src="https://ajax.googleapis.com/ajax/libs/jquery/1.6.2/jquery.min.js" type="text/javascript" charset="utf-8"></script>

4.Include the jCarrotCell.js file after the jQuery script.


<script src="js/jCarrotCell.js" type="text/javascript" charset="utf-8"></script>

5. Finally, call the jCarrotCell function on the element of class ".carrotCell"


<script type="text/javascript">
$(document).ready(function(){
	$('.carrotCell').carrotCell();
});
</script>

Optional. You can also enable swipe support for the carousel on touch devices (iPhone, iPad) simply by including the jquery-touchwipe.js by Andreas Waltl. This enables you to swipe on the carousel to advance forward or back.


<script src="js/jquery-touchwipe.js" type="text/javascript" charset="utf-8"></script>

Horizontal Scroller

The default carrotCell is horizontal and advances by how many is visible in the current view port. See the home page for setup instructions and check the documentation on other setup parameters you can use.

prev
next

The default carrotcell needs to be invoked on the .carrotCell container to start doing all its magic.


$("#defaultKitten").carrotCell();

Make sure you have styles for the .carrotCell container and its child elements, including a fix width view port and fixed same width cell elements.


// insert your own styling for the carousel elements
// including .carrotCell and .carrotCellView width and height
// and .carrotCellView li width and height - these must all be the same size for the calculations to work
// also include the placement of the previous and next buttons
// below are examples
	
#defaultKitten { width: 560px; }
.carrotCellView li { ... } 
.carrotCellView li img { ... }
.carrotCell .next, .carrotCell .prev {
	position: absolute;
	top: 60px;
	display: none;
	cursor: pointer;
	...
}
...

The basic structure of the carrotcell HTML must be followed exactly in order for it to work correctly in basic setup. See the Layout Flexibility demo to find out how to change this mark up.

In the following example, the id of .carrotCell can be changed, as well as the contents of the li and .prev and .next, but nothing else.

Vertical Scroller

This carrotCell scrolls vertically instead of horizontally.

prev next

For a vertical carrotCell you must set the parameter sideways to be the boolean false


$('#defaultVerticalKitten').carrotCell({ sideways: false });

You must create the styles to lay out the .carrotCell and children correctly, this is not included in the .carrotCell.css.


#defaultVerticalKitten { width: 180px; }
#defaultVerticalKitten .carrotCellView { height: 260px; }
...

The mark up used in the vertical carrotCell is exactly the same as the horizontal one, the only difference is your changes in the CSS.

Fall Back State (No Javascript)

This version uses the overflow:auto property of the .carrotCellView container when Javascript is not available. You can also over ride it to just be really long if you wish.
