Previous & Next

If you follow the default mark up, you do not need to do anything special in order for previous and next to work except to style them appropriately in the CSS.

The following example shows how to move the previous and next buttons out of .carrotCell if your markup and styling needs it.

When invoking carrotCell on this mark up, you need to tell it that the .carrotCell container does not containsControl and set it to false in boolean, then also pass in the previous and next selectors so javascript can find where they are. See the documentation for more details.


$('#prevNextKitten').carrotCell({
	containsControl: false,
	prevSelect: "#prevNextExample .controls .fancyPrev",
	nextSelect: "#prevNextExample .controls .fancyNext"
});

Keyboard Navigation

By default the left and right arrow keys control horizontal carrotCells and up and down control vertical carrotCells.

Use the left and right arrow keys to navigate the following example.

All you have to do is set key to the boolean true in the carrotCell params. Note that since key events are a global document event, be careful when binding them to control multiple things.


$('#keyKitten').carrotCell({ key: true });

In this example we'll assign P and Q to be our forward and backward key events instead of arrow keys.

Pass in the corresponding javascript key code of the key you want to assign.


$('#keyCustomKitten').carrotCell({ 
	key: true,
	keyBack: 81, // Q key
	keyForward: 80 // P key
});
