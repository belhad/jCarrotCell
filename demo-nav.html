Previous & Next

If you follow the default mark up, you do not need to do anything special in order for previous and next to work except to style them appropriately in the CSS.

The following example shows how to move the previous and next buttons out of .carrotCell if your markup and styling needs it.

When invoking carrotCell on this mark up, you need to tell it that the .carrotCell container does not containsControl and set it to false in boolean, then also pass in the previous and next selectors so javascript can find where they are. See the documentation for more details.


$('#prevNextKitten').carrotCell({
	containsControl: false,
	prevSelect: ".controls .fancyPrev",
	nextSelect: ".controls .fancyNext"
});

Another way to do the same as above is to set the controlScope, the element in which the controls are contained instead of setting the specifics in the selector. If controlScope is not set, it is set by default to the BODY of the document.

Note that controlScope is assumed to house all of the carrotCell controls including previous, next, pause, play, stop and navi.


$('.example #prevNextKitten').carrotCell({
	containsControl: false,
	controlScope: ".controls",
	prevSelect: ".fancyPrev",
	nextSelect: ".fancyNext"
});

Keyboard Navigation

By default the left and right arrow keys control horizontal carrotCells and up and down control vertical carrotCells.

Use the left and right arrow keys to navigate the following example.

All you have to do is set key to the boolean true in the carrotCell params. Note that since key events are a global document event, be careful when binding them to control multiple things.


$('#keyKitten').carrotCell({ key: true });

In this example we'll assign P and Q to be our forward and backward key events instead of arrow keys.

Pass in the corresponding javascript key code of the key you want to assign.


$('#keyCustomKitten').carrotCell({ 
	key: true,
	keyBack: 81, // Q key
	keyForward: 80 // P key
});

Auto-Generated Navi

You can ask the carrotCell to auto generate a paging control for you. The number of pages is automatically how many times the carrotCell needs to scroll before reaching the end. (Based on the number of items, how many of these items are visible in the view port, and how many items you are moving at once.)

When setting up the carrotCell, pass in the parameters navi and makeNavi in order to auto generate the paging control.


$('.example #naviKitten').carrotCell({ 
	navi: true,
	makeNavi: true
});

Make sure you add in the navi html container inside the .carrotCell element. The following will generate an ordered list (OL) of numbers by default for the paging control.     By default the paging controls are set to be numbers, but you can pass in custom names through the data attribute navi of .navi.

      In this example we only pass in 2 names, and the 3rd page is using the default number name. If you pass in more than enough items they will be ignored.

      Note that data-navi is inclosed in single quotes as per JSON specification and contents of the array in double quotes. Also note that you can not use single or double quotes in the names. For more versatility use the custom navi explained in the next section.

      

        
<ol class="navi" data-navi='["fabulous kitten set", "basket kitten set"]'></ol>
