Configurations

Example of a carrotCell object with configurations:

$('#myCarousel').carrotCell({
     sideways: false,
     delay: 3000,
     key: true,
     navi: true,
     step:1,
     auto: true
});

The configuration options are passed into the carrotCell as an object on creation.

Property Name Property Type Example Description Default
next string "#nextThing"

Control to advance the carousel to the next page.

It can be a class or an id or some other kind of jQuery string selector.

This element must be contained in the ".carrotCell" container.

 ".next"
prev string "#previousThing"

Control to advance the carousel to the previous page.

It can be a class or an id or some other kind of jQuery string selector.

This element must be contained in the ".carrotCell" container.

 ".prev"
off string (class name) ".off"

The class added to previous and next when no action is possible.

This is added to the controls when the end of the list of items is reached.

 ".disabled"
step integer 5

How many items to advance by or how many items a "page" contains

If nothing is set for step, the page is assumed to be all visible elements in the container.

This number should be smaller than the number of total items.

 Number of items visible
sideways boolean true

Make the carousel orientation horizontal if set to true.

 false
infinite boolean true

Make the carousel scroll in circular, never ending motion when set to true.

 false
delay integer (milisecs) 9000

The time delay between auto scrolling of each page.

 5000
speed integer (milisecs) 9000

The speed at which the carousel scrolls.

 1000
navi boolean true

Indicate external navigation is included.

The number of navi items must correspond to the number of pages used.

 false
key boolean true

Turn on or off arrow key navigation for carousel.

Up and Down arrows work for vertical carousels, and Left and Right arrows for horizontal.

Note that since key events are document based all carousels would recognize the event.

 false

API Methods

First retrieve the carrotCell API from the carousel created

var myCarrot = $('#someCarrotCell').carrotCell(); // save the created carrotCell
var myAPI = $(myCarrot).data('carrotCell'); //retrieve the API
myAPI.move(4); // call a method of the API

The configuration options are passed into the carrotCell as an object on creation.

Method Name Parameters Example Description
init options (object) api.init(options);

Initialize the carrotCell with the configuration options object
move pageNumber (integer) api.move(2);

Move to page n of the carousel
insert item (string), index (integer) api.insert('<li>item</li>', 4);

Insert the new carousel element at a certain index.

If the index option is left out, the element is added at the end.
remove index (integer) api.remove(4);

Remove the carousel item at position 4

HTML & CSS

1. .carrotCell contains a single carrotCell and all its components.

2. .carrotCellView class is the view area of the carousel, clipping the extra elements of the content list from being seen.

3. Parent container of the content list, usually an OL or UL.

4. Individual container of the content list, usually a LI.

5. The previous and next buttons are inside .carrotCell but not .carrotCellView, assign styles to .disabled to hide the buttons when they are not operational. By default they are positioned absolutely relative to .carrotCell.

6. Play, Pause, Stop buttons of the auto advance.

7. .navi element contains items corresponding to the pages of the carrotCell.
